Section 01
What is Meshing
  • Process of dividing CAD geometry into small elements (nodes + elements)
  • Converts continuous geometry → discrete numerical model
  • Basis for solving FEA/CFD equations (FEM/FVM)
Section 02
Why Meshing is Important
Accuracy
Controls Result Quality
Directly controls accuracy of stress, temperature, velocity, and pressure results.
Convergence
Solver Stability
Affects solver convergence and stability — poor mesh can cause simulation to diverge.
Performance
Compute Cost
Directly impacts simulation time and computational cost. Over-mesh = slow; under-mesh = wrong.
Resolution
Gradient Capture
Determines resolution of gradients — stress concentrations, temperature spikes, velocity profiles.
Section 03
Effects of Bad Meshing
  • Inaccurate results — false stress concentrations, wrong flow patterns
  • Non-convergence and solver crashes
  • Excessive runtime (over-meshed) or unreliable results (under-meshed)
  • Skewed/distorted elements cause numerical errors
  • Misleading "good-looking" results — garbage in, garbage out
⚠
A simulation can produce smooth, plausible-looking contour plots even with a terrible mesh. Always run a convergence study before trusting results.

Section 5A
Element Shapes

2D Elements

3 NODES
Triangle
Fits any shape, used in free/unstructured mesh. Lower accuracy per element.
4 NODES
Quadrilateral
Higher accuracy, used in mapped/structured mesh. Preferred for shell models.

3D Elements

4 NODES
Tetrahedron (Tet)
Fills any complex volume, easy auto-mesh. Needs more elements for same accuracy as hex.
8 NODES
Hexahedron (Hex)
Best accuracy and efficiency. Requires sweepable or blocky geometry.
6 NODES
Wedge / Prism
Triangular cross-section extruded. Used in inflation/boundary layers.
5 NODES
Pyramid
Transition element between tet ↔ hex mesh regions.

Section 5C
Structured vs Unstructured
Structured
Regular grid (i,j,k), all hex. High accuracy at low element count. Hard to generate for complex shapes.
Unstructured
Irregular connectivity, tet-based. Easy auto-generation, but requires more elements for same accuracy.
Hybrid (Common Practice)
Hex/structured in bulk regions + tet/unstructured in complex zones. Best of both worlds — used in most real-world FEA and CFD models.
Section 5D
Element Order
Linear (1st Order)
Faster, fewer nodes. Less accurate for bending/curved stress. Preferred for CFD.
Quadratic (2nd Order)
Midside nodes for curved edges. Better accuracy for stress/curvature. Higher DOF cost. Preferred for FEA structural.
⚠
Avoid linear tet for bending-dominated structural parts — they are known for stiffness errors (over-stiff behaviour).
Section 06
Fundamentals of Meshing
Global Controls
Uniform settings applied to entire model. Easy and fast to set up. Good starting point for coarse baseline.
Local Controls
Targeted refinement at critical zones only. Far more efficient — refines only where needed.

Workflow

Global coarse-medium mesh→ Identify critical regions→ Apply local controls→ Convergence check→ Finalize
✦
Never refine globally to capture a local effect. Use local sizing instead — global fine mesh wastes enormous compute.
Section 07
Global Mesh Controls
Relevance Center
Coarse / Medium / Fine
Coarse: fast, low accuracy. Medium: default. Fine: high accuracy, high cost for final/critical analysis.
Smoothing
Low / Medium / High
Repositions nodes to improve element quality. Higher smoothing = better quality but more processing time.
Transition
Fast / Slow
Controls growth rate from fine to coarse. Slow = gradual and smoother. Fast = abrupt, possible quality issues.
Span Angle Center
Coarse / Medium / Fine
Controls how finely curved surfaces are meshed. Fine = 36°–12° per element. Mandatory for fillets and circular geometry.
ℹ
For curved/cylindrical parts (bolts, pipes, fillets) — Span Angle Center = Fine is mandatory for accurate stress/flow capture.
Section 08
Local Mesh Controls
Sizing
Body / Face / Edge / Vertex
Controls element size at different scoping levels. Body = entire part. Vertex = point singularities and sharp corners.
Inflation
CFD & Thermal
Thin layered elements near walls to capture boundary layer. Critical for y+ value targeting in turbulence models.
Contact Sizing
Interface Refinement
Fine mesh at contact interfaces (bonded, frictional, sliding) for accurate load transfer. Mismatched sizes = inaccurate pressure.
Refinement
Post-Mesh Subdivision
Subdivides existing elements in a region without full remesh. Levels 1–3, each roughly doubles density.

Section 14
Body Sizing

Behavior: Soft vs Hard

Soft (Recommended Default)
Size is a target/max. Mesher can refine finer if other controls (face sizing, contact sizing, curvature) demand it. Smooth blending between regions.
Hard (Use with Caution)
Size forced uniformly. Ignores and overrides finer local controls on same body. Can cause size conflicts if combined with smaller local sizing.

Workflow

  • Select target body
  • Uncheck "Default" on Element Size → enter required size
  • Keep Behavior = Soft (recommended)
  • Use Hard only if uniform size is mandatory for solver/element count requirement
  • Adjust Defeature Size if body has small features to suppress
Sections 15–17
Face / Edge / Contact Sizing
Face Sizing
Surface Control
Controls mesh on specific surfaces. Good for thin faces, flow inlets. Soft/Hard behavior same as body sizing.
Edge Sizing
Edge Divisions
Type: Element Size or Number of Divisions. Use "Number of Divisions" for consistent mesh across symmetric features.
Contact Sizing
Interface Mesh
Scoped to contact regions. Always match mesh size on both sides of a contact pair for accurate pressure distribution.
Section 18
Inflation (Detailed)
First Layer Thickness
Most precise y+ control. Calculate first layer height from y+ formula based on turbulence model.
Smooth Transition
Convenient auto-sizing. Less precise y+ control but easier to set up. Good default starting point.

Inflation Workflow

Calculate y+→ Compute first layer height→ Set Max Layers 10–20→ Growth Rate 1.1–1.2→ Algorithm = Pre→ Generate + Verify visually
✕
"Boundary = No Selection" — no inflation will be generated. You must select wall face(s) before running.
Section 19
Repair Topology

Automatically detects and fixes CAD defects (thin faces, short edges, sharp angles, small holes) before meshing. Used on dirty/imported CAD causing mesh failures.

Merge Faces
Merges adjacent small/coplanar faces into one larger face.
Suppress Edges
Ignores insignificant edges — edge-level defeaturing.
Remove Thin Faces
Removes narrow/sliver faces causing bad elements.
Collapse Short Edges
Collapses edges below a length threshold into a point.
Remove Sharp Angle Faces
Removes faces creating very sharp angles with neighbors.
Fill Hole
Fills small irrelevant holes/gaps in geometry.
⚠
Enable only what's needed. Enabling everything can oversimplify geometry and remove important features. All options default to "No" — must be explicitly enabled.
Section 20
Quad Layer

Generates a structured "washer" ring of quad elements around circular holes/edges. Used for bolt holes, pin holes, circular cutouts in shell/plate FEA models.

Key Parameters
Min Divisions: 6 (bump to 8–12+ for bolt-hole accuracy)
Washer Layers: 1 (increase for smoother blend into field mesh)
Direction: Program Controlled
Workflow
1. Select circular edge(s)/hole(s) → Geometry (mandatory)
2. Set Washer Layers >1 for smooth transition
3. Pair with Edge Sizing on hole edge for circumferential size control

Jacobian Ratio — Deep Dive

Solver maps each element from an ideal reference shape to its real distorted shape using the Jacobian matrix. Ratio = max/min of the matrix determinant across integration points. Critical failure: if determinant goes negative/zero → element has inverted → fatal solver error.

Main Causes
Coarse mesh on highly curved surfaces · Sharp/acute angle elements · Poor transition zones · Quadratic elements on curved edges with insufficient divisions
Fixes
Refine mesh on curved regions (lower Span Angle Center) · Switch to Linear locally · Improve geometry (Pinch/Repair Topology) · Use Slow transition near curvature

Quality Check Workflow

Generate mesh→ Set Mesh Metric→ Check histogram min/worst→ Identify flagged elements→ Local fix + remesh
⚠
Always check worst-case element, not average. One bad element can crash a solver. CFD: prioritize Orthogonal Quality + Skewness; ignore high Aspect Ratio in BL regions.
Section 22
Mesh Convergence & Optimization

Convergence Study Workflow

Baseline mesh → record result→ Refine mesh→ Re-run → compare % change→ Repeat until <2–5% change→ Use converged mesh

Adaptive Meshing

h-Adaptivity (FEA)
Refines element size based on error estimator in Mechanical static structural. Auto-refines high-gradient regions between iterations.
Solution-Based (CFD)
Refines based on result gradients/y+ in Fluent CFD. Cuts manual remeshing effort for unfamiliar geometry.
✦
Sharp re-entrant corners/point loads → stress never converges (singularity). Focus elsewhere or use averaged stress. Always cap max refinement levels — unconstrained adaptive can explode element count.

Performance Tips

  • Local sizing > global fine mesh — always
  • Use symmetry / 2D simplification where valid
  • Linear elements unless bending/curvature accuracy needed
  • Hex > Tet where geometry allows (fewer elements, same accuracy)
  • Minimum inflation layers for required y+ — not more
  • Defeature irrelevant small features (Pinch/Defeaturing)
  • Debug BCs and contacts on coarse mesh first, finalize on refined mesh
  • Use distributed/parallel solve for large models
Section 23
The "Perfect Mesh" Checklist
  • ✓ No errors/warnings in mesh generation log
  • ✓ Element Quality avg >0.3 (FEA) — no elements near 0
  • ✓ Jacobian Ratio clean — no negative/folded elements
  • ✓ Aspect Ratio <5–10 outside boundary layers
  • ✓ Skewness <0.8, Orthogonal Quality >0.1–0.15 (CFD critical zones)
  • ✓ Inflation captures target y+ (CFD)
  • ✓ Contact faces have matching/compatible mesh sizes
  • ✓ No free/disconnected/duplicate elements
  • ✓ Element count fits compute budget
  • ✓ Visual check — no gaps, overlaps, or distorted regions
